# 快速启动指南

## 🚀 三步开始使用

### 1️⃣ 安装依赖
```bash
npm install
```

### 2️⃣ 启动应用
```bash
npm run dev
```
应用将在 `http://localhost:8080` 启动并自动打开浏览器。

### 3️⃣ 首次设置

1. **设置主密码**
   - 应用启动后，输入一个强密码（至少8个字符）
   - 再次输入确认密码
   - 点击"设置密码"
   - ⚠️ 请牢记密码，遗失后无法恢复数据

2. **开始使用**
   - 点击"📁 导入文件"添加图片或视频
   - 使用侧边栏创建类别来组织文件
   - （可选）配置AI服务进行自动打标

## 📋 主要功能

### ✅ 已实现功能

- ✅ **加密存储** - AES-256-GCM 端到端加密
- ✅ **文件导入** - 支持图片和视频（拖拽/选择）
- ✅ **类别管理** - 创建、编辑、删除类别
- ✅ **网格预览** - 缩略图网格，显示分辨率
- ✅ **视频支持** - 自动提取关键帧
- ✅ **AI打标** - OpenAI兼容API自动标注
- ✅ **批量操作** - 批量导入、打标、删除
- ✅ **详情查看** - 查看完整图片/视频和标签
- ✅ **密码锁定** - 随时锁定应用保护隐私
- ✅ **设置管理** - AI配置、通用设置

### 🔧 配置 AI 打标（可选）

1. 点击右上角"⚙️ 设置"
2. 在"AI配置"标签页填写：
   ```
   API URL: https://api.openai.com/v1
   API Key: sk-your-api-key-here
   模型名称: gpt-4o
   ```
3. 点击"测试连接"验证
4. 点击"保存配置"

支持的API：
- OpenAI (GPT-4o, GPT-4-vision)
- 任何OpenAI兼容的API服务

## 💡 使用技巧

### 快捷键
- `Ctrl/Cmd + I` - 导入文件
- `Ctrl/Cmd + L` - 锁定应用
- `Ctrl/Cmd + 点击` - 多选文件
- `ESC` - 关闭模态框

### 批量选择
1. 按住 `Ctrl/Cmd` 键
2. 点击要选择的文件
3. 点击"🗑️ 批量删除"执行操作

### AI批量打标
1. 选择一个类别（或"全部"）
2. 点击"🏷️ AI批量打标"
3. 系统会自动标注所有未打标的文件
4. 每个文件间隔1秒，避免速率限制

## 🔒 安全说明

### 数据加密
- 所有文件在存储前都会加密
- 使用Web Crypto API标准加密
- 密钥由密码派生，从不存储

### 隐私保护
- 数据完全本地存储（IndexedDB）
- 不上传任何内容到云端
- AI打标仅发送图片内容（可选功能）
- 锁定后密钥从内存清除

### 密码管理
- **重要**: 密码遗失后无法恢复
- 建议使用密码管理器保存
- 密码强度：至少8字符，建议包含大小写、数字、符号

## 📊 性能建议

- 单次导入建议不超过 **100个文件**
- 单个文件建议不超过 **500MB**
- 大量文件时使用类别分组
- 定期清理不需要的文件

## ❓ 常见问题

### Q: 忘记密码怎么办？
A: 密码无法恢复。需要清除浏览器数据并重新设置。

### Q: 数据存储在哪里？
A: 存储在浏览器的IndexedDB中，不同浏览器数据独立。

### Q: 可以导出数据吗？
A: 目前可以导出元数据（JSON），文件内容暂不支持导出。

### Q: AI打标消耗多少费用？
A: 取决于使用的API服务。GPT-4o约为每1000张图片 $10-20。

### Q: 支持哪些文件格式？
A:
- 图片: JPEG, PNG, GIF, WebP
- 视频: MP4, WebM, OGG, MOV, AVI, MKV

### Q: 如何迁移到其他电脑？
A: 目前需要在每台电脑独立使用。未来版本会支持导出/导入。

## 🐛 故障排除

### 应用无法启动
1. 确保使用现代浏览器（Chrome, Edge, Firefox最新版）
2. 检查浏览器控制台是否有错误
3. 清除浏览器缓存后重试

### 文件导入失败
1. 检查文件大小是否超过500MB
2. 确认文件格式是否支持
3. 尝试单个文件导入测试

### AI打标不工作
1. 检查API URL和API Key是否正确
2. 点击"测试连接"验证配置
3. 查看浏览器控制台错误信息
4. 确认API账户有足够额度
